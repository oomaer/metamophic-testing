# SMTP State Reset Metamorphic Relation

## Overview
This test validates RFC 5321 RSET command behavior - the server must completely reset its state after RSET, as if starting a fresh transaction.

**Metamorphic Relation:**
- Connection A: Perform full SMTP transaction
- Connection B: Perform partial transaction, send RSET, then same full transaction
- Expected: Both connections should receive identical responses

## Files

- `grammar.fan` - Fandango grammar defining the SMTP state reset metamorphic relation
- `script.py` - Main test script that runs Fandango and captures violations
- `run_test.py` - Test runner that manages SMTP server lifecycle
- `generate_all_configs.py` - Generates colorized summary report from test logs
- `test_output.log` - Generated log file with detailed test results
- `all_test_cases.txt` - Generated summary report with all test case details

## Usage

### Quick Start (Recommended)
Run the complete test suite with automatic server management:
```bash
python run_test.py
```

This will:
1. Kill any existing test servers
2. Start SMTP servers on ports 8026 and 8027
3. Run the metamorphic tests
4. Display summary with violations/passes
5. Clean up servers automatically

### Manual Run
If you want to run just the test script (servers must be running):
```bash
python script.py
```

### Generate Summary Report
After running tests, generate a detailed colorized summary:
```bash
python generate_all_configs.py
```

This creates `all_test_cases.txt` with:
- Summary table of all test cases
- Detailed view of each test case (Connection A vs B)
- Reset path breakdown
- Final compliance summary

## Mutation Testing

To test with different mutants, edit the `mutant_id` in:
```
/Users/i7949486/Desktop/uds/thesis/mutation/aiosmtpd/run-test-server.py
```

Available mutants:
- `NONE` - Baseline (correct behavior)
- `STRICT_DUP_REJECT` - Rejects duplicate recipients
- `CASE_INSENSITIVE_REJECT` - Case-insensitive duplicate checking
- `MAX_RCPT_LIMIT_2` - Enforces max 2 recipients
- `REVERSE_LOGIC` - Inverted acceptance logic

## Troubleshooting

**Port already in use:**
```bash
pkill -f run-test-server.py
```

**Test hangs:**
- Check if servers are responding: `telnet localhost 8026`
- Increase timeout in `run_test.py` (default: 300s)

**No violations detected:**
- Check grammar constraints in `grammar.fan`
- Verify servers are returning proper SMTP codes
- Review `test_output.log` for detailed execution flow
